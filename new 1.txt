/tablerolls/:charId			get /api/sheet/charId/tableroll/ and the char.selectedPerks and diplay the list of table rolls, with each table name being a 
							link to /tablerolls/:charId/:rollId/:tableId. the list of already selected perks is below
	controller: 
		$scope.tableRolls
		$scope.selectedPerks = sheet.selectedPerks
	for roll in tableRolls
		for table in roll
			<a href=''> {{table.displayName}} </a>
			
	for perk in selectedPerks
		{{perk.description}}
		
/tablerolls/:charId/:rollId/:tableId		get the list of perks for this table, with the ones the sheet already has greyed out somehow. show a 
											roll button to select a random perk. when the new one's selected, just show the result, and a back 
											button that goes to /tablerolls/:charId
	controller: 
		$scope.table
	for perk in table
		if (selected)
			grey out
		{{perk.range}}  {{perk.description}}
	<button> roll </button>
					
/api/sheet/charId/tableroll/	get the sheet's available rolls, as array of tablerolls with arrays of tables
/api/table/tableId/charId/		get the sheet's options for the given table (the table perks, with the already selected ones marked)
/api/sheet/charId/tableroll/tableId/rollId		post to make the roll on this table for the given tableroll, returning the selected perk


/api/sheet/charId/tableroll/
	Sheet sheet = getsheet()
	List<> rolls = sheet.getBenefitTableRolls();
	return rolls?
	
/api/table/tableId/charId/
	look up the benefit table by tableId
	look up the sheet's chosen perks
	for each chosen perk 
		flag the perk in the table as chosen
	return table
	
/api/sheet/charId/tableroll/tableId/rollId			POST
	load the table
	make sure there is at least one 
	roll d20, choose perk from range
	if (perk is already selected)
		reroll
	add perk to the sheet's selected perks, for table roll rollId
	if (all rolls are made) 
		flag sheet as completedTableRolls
	return selected perk
	
	



when finishing the table roll phase, set hasCompletedTableRoll. 
when finishing the table roll phase, or after swapping two perks, check if there are two or more unswapped perks, if not, set hasCompletedTableRollSwaps

once roll picks are done, go to /tablerollswap/:charId, show selected perks with checkboxes, the tables, a swap button and a finish button. swap button goes to the table's perk choices, where you can choose any you don't already have. the finish button completes the swap step.
